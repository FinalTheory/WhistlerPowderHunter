------

Task: DECISION_TASK

Inputs:
- Outputs/signals from other selected tasks.
- RWDI synopsis and provided model/sensor context already in conversation.
- Recent observed snowfall statistics and recent alpine lift/opening history.
- HRDPS hourly frames from latest run time to next local 11:00 (Whistler PST/PDT) are already provided for DECISION_TASK in the current conversation context.

Goal:
Synthesize already-selected weather-task signals into explicit powder-oriented tactical decisions, rather than generic skiability commentary. Provide clear tactical decisions for:
1) tomorrow daytime,
2) day-after-tomorrow daytime,
and briefly indicate whether a better powder window is likely after that (without requesting out-of-window data).

Execution rules:
- Keep all decisions inside existing `summary.en` and `summary.zh`.
- In TL;DR, include explicit labels:
  - Tomorrow: GO / GO_WITH_RISK / NO_GO
  - Day After Tomorrow: GO / GO_WITH_RISK / NO_GO
  - Next Better Window: YES / NO (add short note if YES)
- Format these three TL;DR lines as "Conclusion + short note" in a fixed style:
  - English:
    - Tomorrow: <STATUS> (<short reason>)
    - Day After Tomorrow: <STATUS> (<short reason>)
    - Next Better Window: YES/NO (<short note>)
  - Chinese:
    - 明天：<STATUS>（简短原因）
    - 后天：<STATUS>（简短原因）
    - 下一个更好窗口：YES/NO（简短备注）
- In each line, put the conclusion first and the note in parentheses after it.
- Keep each note short and focused on the 1-3 most decision-relevant drivers only.
- Use the overnight-to-next-local-08:00 evolution as primary evidence for first-chair timing and tomorrow go/no-go.
- Define the primary ski window for each target day as local first-chair through late morning (roughly 08:00 to about 12:30 local Whistler time), unless the evidence clearly supports a later worthwhile powder window.
- Do not request additional HRDPS frames inside the provided time window unless a specific key frame is missing or unreadable.
- Keep the decision target powder-oriented and tactical. This task is about whether the setup justifies first-chair powder effort for local skiers, not generic skiability or general resort-day enjoyment.

Status semantics:
- GO:
  - broadly worthwhile for powder-oriented effort
  - flexible enough timing that the opportunity does not depend on a fragile or narrow condition
- GO_WITH_RISK:
  - use only when there is a real powder opportunity, but that opportunity depends on a fragile and clearly identifiable condition
  - common examples include a narrow early window, fragile alpine access timing, or fragile preserved carry-over support
  - it is not a generic middle state between GO and NO_GO
- NO_GO:
  - not worthwhile for powder-oriented timing in the primary window
  - this can still include generally skiable conditions; it simply means the setup does not justify tactical first-chair powder effort

Decision logic:
1. Snow support requirement
   - If there is neither meaningful new snowfall support nor credible preserved carry-over powder support, default to NO_GO for tactical powder decisions.
   - If expected snowfall plus any credibly preserved carry-over snow is still unlikely to support roughly 10cm+ of dry-enough snow in ski-relevant terrain by the primary window, default to NO_GO.
   - Do not use GO_WITH_RISK for a small overnight refresh alone unless there is separate credible preserved carry-over powder support, or the latest guidance still shows a credible upside scenario in band placement, persistence, or intensity that could push totals toward roughly 10cm+ by the primary window.
   - Wet or rain-affected snow at mid-mountain does not by itself justify a powder-oriented recommendation.
   - This task is only for powder-oriented first-chair timing, not general resort-day enjoyment planning.
   - Dense wet snowfall, storm-skiing refresh, or merely skiable wet/rain mid-mountain conditions do NOT by themselves qualify as worthwhile powder value.

2. Carry-over preserved powder support
   - Treat recently accumulated but not meaningfully accessed snow as valid carry-over support for later powder decisions.
   - Count carry-over support only when all of the following are reasonably supported:
     - substantial snow has already fallen in ski-relevant terrain
     - alpine access was materially limited or closed during that snowfall, or is very likely to remain materially limited or closed while that snowfall is occurring
     - temperature / phase / settlement risk still support preservation into the next target primary ski window
   - For tomorrow: if today's carry-over support is strong and tomorrow's operational risk is low enough for broad alpine access, that alone can justify GO even if additional overnight snowfall is limited.
   - For tomorrow: if the preserved opportunity still looks worthwhile but depends on fragile access timing or fragile preservation into tomorrow's primary ski window, use GO_WITH_RISK instead of GO.
   - For day-after-tomorrow: if the main snowfall event is expected tomorrow and tomorrow itself carries substantial operational risk, do not assume the day-after outcome is already GO. Instead, state explicitly in the note that day-after-tomorrow could upgrade to GO if tomorrow's alpine mostly stays closed and the snow is preserved well enough.
   - For `Next Better Window`: if the main snowfall event occurs tomorrow, day-after-tomorrow, or both, and operational risk looks high across those storm days, `Next Better Window` may be YES even without requiring a fresh new snowfall cycle afterward. In that case, say clearly that the upside comes from likely carry-over snow preserved by alpine closure rather than from a new storm.
   - Do not upgrade on this basis if warming, rain, settlement, crusting, or continued operational restriction is likely to materially degrade the opportunity before or during the relevant primary ski window.

3. Wind-first operational constraint
   - If strong wind is likely to shut or severely limit alpine through most of the primary ski window, default to NO_GO unless there is clear evidence of a short early alpine-access window that still preserves worthwhile powder quality.
   - If wind risk is elevated but likely improves early enough to preserve a meaningful part of the primary ski window, use GO only when the opportunity remains broadly worthwhile without a fragile timing dependency; otherwise use GO_WITH_RISK with an explicit timing caveat.

4. Warming / phase-degradation override
   - Treat warming / phase degradation as a hard negative override for this task.
   - Return NO_GO if the main ski window is likely already compromised by any of the following:
     - freezing level near or above alpine elevation ~2000m during active precipitation
     - 546 dam thickness line sitting over or very near Whistler during the main precipitation window, as part of a broader warm / mixed-phase signal
     - clear rain / mix risk in upper-mountain terrain
     - sensor evidence showing near or above freezing temperatures at key ski elevations such as Roundhouse, Rendezvous, Symphony, Harmony, or Pig Alley during active or imminent snowfall
     - strong wind likely to prevent meaningful alpine access during the primary ski window
   - Use the 546 dam signal as a thermal proxy, not as a one-frame edge-touch trigger by itself.
   - In warm atmospheric-river or pineapple-express-type setups, wet snow, mixed snow, or rain-affected upper-mountain snow should default to NO_GO unless there is unusually strong and specific evidence that the primary ski window still remains cold enough for genuinely worthwhile powder quality.
   - Do not use GO_WITH_RISK when the main ski window is already warm / wet enough that quality is materially degraded before or during the primary ski window.
   - Use GO_WITH_RISK only when there is still a realistic early worthwhile window before degradation or access failure, and that early window is still likely to offer dry-enough powder rather than dense wet storm snow.
   - If an early start is required because of warming, rain, rising snow line, rapid snow-quality degradation, or likely access loss later in the morning, use GO_WITH_RISK only if that early period still offers a genuinely worthwhile powder opportunity. Otherwise use NO_GO.
   - Distinguish "can still score early dry turns" vs "already too warm / wet during main window" explicitly.
   - In warm atmospheric-river / pineapple-express-type setups, do not assume a narrow early-morning salvage window exists just because the setup is borderline. Consider it only as a rare exception when recent decision-relevant evidence from the latest guidance and same-day observations explicitly supports it.
   - If such a window is clearly supported by that recent decision-relevant evidence, use GO_WITH_RISK rather than NO_GO, and explain the fragile timing constraint clearly.
   - Do not use that salvage-window exception if wind / operational evidence still strongly indicates that meaningful alpine access is unlikely during the same early window; in that case, keep NO_GO.
   - If such a window is not clearly supported by that recent decision-relevant evidence, say so clearly and use NO_GO.
   - Do not use GO_WITH_RISK merely to soften an already compromised warm / wet outcome. If the primary ski window is materially degraded before or during the main powder window, use NO_GO.

5. Priority order
   - Warming / phase-degradation override > operational wind / access constraint > snowfall support (including preserved carry-over powder).
   - If recommendation could flip, state the flip condition explicitly (for example, faster wind drop or faster warm up or stronger overnight persistence).

Notes:
- Keep decisions concise and actionable.
- Keep uncertainty explicit and tied to specific variables.
- If a hard negative override is met, do not hedge back toward GO_WITH_RISK merely because raw snowfall amounts are large.
- Respect the global time window: tomorrow daytime, tomorrow night precipitation, and up to day-after-tomorrow daytime.
- Do not request additional images outside this window; if a better window likely exists later, mention it briefly with low-confidence wording.
- Do not provide backcountry avalanche travel guidance.
